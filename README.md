# WC2026 Office Sweepstake

Live leaderboard for an office World Cup 2026 sweepstake. Static site (GitHub
Pages) + a scheduled GitHub Action that pulls results from
[football-data.org](https://www.football-data.org/) — no manual result entry.

## How it works

- 48 teams are split into **6 tiers of 8** by FIFA world ranking.
- Every participant gets **one random team per tier** (works for any headcount;
  with more than 8 people teams have multiple owners).
- The draw is deterministic from a recorded seed → auditable, no re-rolls.
- A GitHub Action runs every 30 minutes on tournament days, writes
  `data/results.json`, and the page recomputes the leaderboard from it.

## Files

| Path | What |
|---|---|
| `index.html` | The whole site (leaderboard, draw, matches, awards, rules) |
| `data/participants.json` | Names + draw seed — **edit this**, then re-run the draw |
| `data/draw.json` | Generated by `scripts/draw.mjs` |
| `data/results.json` | Written automatically by the Action |
| `data/extras.json` | Optional: red-card counts per team id for the Red Mist award |
| `scripts/draw.mjs` | Seeded draw generator |
| `scripts/fetch-results.mjs` | football-data.org → `results.json` |

## One-time setup

1. **API key**: register (free) at
   <https://www.football-data.org/client/register>, then add the key as a repo
   secret named `FOOTBALL_DATA_TOKEN` (Settings → Secrets and variables →
   Actions).
2. **GitHub Pages**: Settings → Pages → deploy from branch `main`, root.
3. **Run the real draw** once sign-ups close:
   - Put the final names in `data/participants.json`, set `"sample": false`,
     and (optionally) change the seed — e.g. include the date so it's clearly
     fixed in advance.
   - `node scripts/draw.mjs`
   - Commit & push. Done — the draw page updates.
4. Optionally trigger the Action manually (Actions → Update results → Run
   workflow) to pull the first fixtures.

## Scoring

Group: win 3 / draw 1, +1 per goal scored (all stages, excluding shoot-outs).
Knockout wins: R32 +4, R16 +6, QF +8, SF +12, third-place play-off +4,
Final +16. Penalties count as a win. Tiebreak: combined GD, then goals scored.

npx serve          # or: python -m http.server
